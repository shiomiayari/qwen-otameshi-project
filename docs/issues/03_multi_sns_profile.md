# Issue: 複数SNS連携 & 簡易プロフィールカード (Link-in-Bio) の作成

*   **Type:** Feature (新機能)
*   **Labels:** `feature` 🆕, `profile-card` 🔗, `networking` 🤝
*   **Priority:** High

---

## 概要 (Overview)
現在のアプリは単一のSNS（XまたはInstagramのいずれか）のQRコードしか発行できません。これを発展させ、複数のSNSリンク（X、Instagram、GitHub、自己紹介テキスト等）をまとめた「自己紹介プロフィールページ」をアプリ内に作成し、QRコードがそのプロフィールページを指すように変更します。

これにより、イベント参加者同士がQRコードを1回読み取るだけで、すべての主要なSNSアカウントを一度に交換できるようになります。

## ユーザー体験 (User Experience)
1. トップページで名前を入力し、SNS登録画面へ進む。
2. X、Instagram、GitHubなどの複数のURL入力欄と、短い自己紹介文（任意）の入力欄が提供される。
3. 発行されたパスのQRコードは、個別のSNSではなく `/profile?data=...`（またはID）を指すようになる。
4. 他の人がスマホでスキャンすると、美しいLinktree風のプロフィール画面が立ち上がり、各SNSのリンクアイコンから直接フォローできるようになる。
5. 「この人の連絡先を保存」ボタンを押すことで、アプリ内の「交換履歴リスト」に保存できる。

## 実装要件 (Requirements)

### 1. 入力フォームの拡張
*   [ ] 複数のSNS URL（X, Instagram, GitHubなど）を入力できるフォームコンポーネントを設計。
*   [ ] 各種URLの形式バリデーション（簡易チェック）。

### 2. プロフィール閲覧画面の作成 (`src/app/profile/page.tsx`) [NEW]
*   [ ] クエリパラメータ（例: `?name=ayari&x=...&ig=...`）からデータを読み取り、自己紹介カードを動的にレンダリングする。
*   [ ] 各種SNSへの綺麗なボタンリンク（アイコン付き）を表示する。

### 3. 交換履歴・マイリスト機能
*   [ ] `localStorage` を使用し、スキャンして開いたプロフィールのデータを「マイコンタクト（連絡先リスト）」に自動またはボタン押下で保存。
*   [ ] トップページやメニューから「出会った人リスト」を表示できる履歴画面を作成。

## 開発のヒント (Technical Hints)
*   データベースを使わずに実装するため、プロフィールURL（QRコードの中身）はパラメータを長く連結させるか、入力データをBase64やJSON文字列でエンコードしてURLに埋め込みます（例: `/profile?d=eyJuYW1lIjoiYW...`）。
*   URLが長くなりすぎるとQRコードの密度が高くなりスキャンしづらくなるため、Base64圧縮等を適切に行います。
