# Issue: 管理画面：イベント情報設定 & デザインPNGアップロード機能の追加

*   **Type:** Feature (新機能 / 管理者用)
*   **Labels:** `admin` ⚙️, `settings` 🔧, `file-upload` 📤
*   **Priority:** High

---

## 概要 (Overview)
イベント主催者が、イベント名や開催日時、場所などを設定できる「管理画面」を追加します。
また、イベント独自のブランドイメージを表現するため、**名前札の背景やフレーム用のオリジナルデザイン画像（PNG）をアップロードして登録できる機能**を実装します。これにより、発行される参加者のパスカードへ自動的にデザインが反映されるようになります。

## ユーザー体験 (User Experience)
1. 主催者が管理画面（例: `/admin`）にアクセスする。
2. イベント名、日時、場所などの基本情報を入力する。
3. **「ライトモード用デザイン画像（PNG）」**および**「ダークモード用デザイン画像（PNG）」**をアップロードする。
4. 設定を保存すると、一般参加者がアクセスした際のトップページや、発行されるパスカードのデザインが、登録された基本情報とPNG画像に置き換わる。

## 実装要件 (Requirements)

### 1. 管理設定画面の作成 (`src/app/admin/page.tsx`) [NEW]
*   [ ] イベント基本情報入力フォーム（イベント名、日付、場所）。
*   [ ] 背景・フレーム用PNG画像のドラッグ＆ドロップ対応アップロードフォーム。
    *   ライトモード用・ダークモード用の2スロットを用意する。

### 2. デザインデータのローカル保存（フロントエンド完結）
*   [ ] アップロードされたPNG画像を `FileReader` の `readAsDataURL` を使用して Base64 形式に変換。
*   [ ] イベント基本情報と Base64 化された画像データを、まとめてブラウザの `localStorage` に保存（またはSessionStorageに保持）。

### 3. 一般参加者画面へのイベント設定・デザインの動的反映
*   [ ] 参加者が名前を入力してパスを生成する際、ローカルストレージからイベント背景PNGを読み出し、パスカードの背景レイヤーとして重ねて描画する。
*   [ ] 画像ダウンロードやサーマルプリンター印刷の際にも、このカスタムPNG背景が合成されたデータで出力されるように統合する。

---

## 開発のヒント (Technical Hints)
*   Base64に変換した画像データはデータサイズが大きくなるため、`localStorage` の上限（一般的に約5MB）を超えないよう、アップロード時に最大サイズ制限（例: 500KB以下）を設定するか、Canvasを用いて解像度を適切なサイズ（例: 幅384px前後のレイアウトに合わせたサイズ）にリサイズ・圧縮した上で保存します。
