# Issue: Web Bluetooth を利用した LX-D02 プリンターでの物理印刷機能

*   **Type:** Integration (外部連携)
*   **Labels:** `integration` 🔌, `hardware` ⚙️, `bluetooth` 📡
*   **Priority:** High

---

## 概要 (Overview)
`lx-printer` ライブラリを使用して、Webブラウザから Web Bluetooth API 経由で **LX-D02 サーマルプリンター**に接続し、発行された受付パス（名前札とQRコード）をその場で物理的にシールや紙として印刷できる機能を追加します。
管理画面で設定されたカスタムデザインPNG背景、お名前、QRコードをリアルタイムに1つの印刷用イメージとして合成・変換し、プリンターへ送信します。

## ユーザー体験 (User Experience)
1. 参加者が受付パス生成画面で「パスを印刷」をタップ。
2. ブラウザのBluetoothペアリング画面が表示され、`LX-D02` デバイスを選択して接続。
3. 画面上に「接続完了」「印刷中... (XX%)」などの状態が表示され、プリンターから名前札がプリントアウトされる。
4. 印刷完了後、自動で切断され、元の画面に戻る。

## 実装要件 (Requirements)

### 1. ライブラリセットアップ
*   [ ] `lx-printer` をプロジェクトの依存関係に追加。
*   [ ] モジュール解決（Next.js / TypeScript の設定）が正しく行えることを確認。

### 2. 印刷用 Canvas の合成ロジック
*   [ ] プリンターの印字領域（幅384px、高さは内容に応じて可変）に対応した印刷専用 Canvas を生成するヘルパー関数を実装。
*   [ ] 描画の優先順位:
    1. 管理画面からアップロードされたカスタム背景PNG（ライト/ダーク選択に応じたもの）
    2. イベント名・日時などのテキストヘッダー
    3. 参加者のお名前（中央揃え、大きめのフォント）
    4. 参加者のQRコード（中央または右下）

### 3. `LXD02Printer` との通信実装

*   [ ] ステータスフィードバックUIの実装（バッテリー残量のパーセンテージ、用紙切れ状態、エラーハンドリング）。

## 開発のヒント (Technical Hints)
*   Web Bluetooth はユーザーのジェスチャー（クリック/タップなど）を契機として呼び出さなければ動作しません（ページ読み込み時の自動接続などはブラウザセキュリティにより禁止されています）。
*   `lx-printer` は画像のFloyd-Steinbergディザリング（モノクロ化）をライブラリ内部で自動処理してくれますが、あらかじめモノクロ印刷で視認性が高くなるように、 Canvas で描画するテキストや境界線のコントラストを高めに設計しておくと綺麗に印刷されます。
