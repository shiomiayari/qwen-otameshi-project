# Issue: パスカードのカスタマイズ（ライト/ダーク）とオフライン画像保存の追加

*   **Type:** Enhancement (改善)
*   **Labels:** `enhancement` 🎨, `ui-ux` 📱, `offline` 💾
*   **Priority:** Medium

---

## 概要 (Overview)
生成される受付パス（名前札）のデザインを、ユーザーが「ライトモード」と「ダークモード」の2種類から選択できるようにし、さらにその名前札カードを画像（PNG）としてスマートフォン等のローカル端末に保存・ダウンロードできるようにします。

## ユーザー体験 (User Experience)
1. パス生成完了画面で、「ライトモード」または「ダークモード」のトグルを切り替えることで、名前札全体の配色が即座に変更される。
2. 「画像として保存」ボタンを押すと、画面上の名前札デザイン（背景、お名前、QRコード）が1つの画像ファイル（PNG）としてダウンロードされる。
3. ユーザーはこれをスマホのカメラロールに保存し、受付時に通信せずに（オフラインで）提示できる。

## 実装要件 (Requirements)

### 1. デザインテーマの切り替え機能
*   [ ] パスカード表示部分に「ライトテーマ / ダークテーマ」の選択トグルUIを実装。
    *   **ライトテーマ:** 白/淡いグラデーション背景、濃い文字。
    *   **ダークテーマ:** 漆黒/深みのあるダークグレー背景、ネオン調グラデーション、白い文字。

### 2. パスカードの画像ダウンロード機能
*   [ ] パスカードの見た目（背景色/画像、お名前、QRコードなど）をHTML5 `<canvas>` 上にレンダリングする処理を実装。
*   [ ] 生成された Canvas を `canvas.toDataURL("image/png")` でPNG形式のデータURLに変換。
*   [ ] アンカータグ (`<a>`) を動的生成し、`download="pass-card.png"` 属性を付与してダウンロードを実行させるトリガーの実装。
*   [ ] スマホブラウザでの利便性向上のため、対応している場合は Web Share API を呼び出してシステム標準の「共有メニュー（画像の保存など）」を呼び出せるようにする。

---

## 開発のヒント (Technical Hints)
*   Canvasに背景画像やQRコード画像を描画する場合、画像の読み込み完了（`onload`）を待ってから描画プロセスを実行する必要があります。
*   CORSの問題を防ぐため、QRコードのDataURL（base64）をCanvasに描画する際はエラーが発生しにくいです。
